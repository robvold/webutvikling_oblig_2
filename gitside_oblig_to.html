  • Git og GitHub

Git er et versjonskontrollsystem. Versjonskontrollsystem er en type programvare som kjennetegnes ved å være distribuert. At det er distribuert betyr blant annet at programmet fungerer gjennom repositorier. Dette betyr igjen at mange brukere kan arbeide på en fil samtidig. Når de gjør dette lages en egen fil på din datamaskin. Du kan så pushe den, altså dytte filen opp mot hovedarbeidet. I tillegg kan du også pulle. Det betyr å dra inn den gjeldende versjonen, så du kan forandre på riktig fil. Git ble lagd av finnen Linus Torvald, som også er skaperen av Linux.
GitHub er en webbasert Git-tjeneste. GitHub er spesiallagd for koding, spesielt med sammarbeid med koding av større prosjekter. Den lar deg, og andre jobbe med samme prosjekt på hver sin egen pc. I tillegg har den den sammen generelle Git-funksjolaliteten, med lagring av tidligere versjoner av filen dere koder sammen. Og at det lar deg trekke tilbake forandringer du har gjort tidligere.
